调试

在制作游戏时，了解特定时间点的游戏机制的某些细节，或者能够为某些功能输入某些键值，以测试游戏的不同方法和想法，这通常非常重要。

GameMaker为此提供以下函数：

  • 一些基本的调试函数，在调试模式下(或在Windows目标上)显示消息并请求输入。当为其他目标模块编译时，这将被忽略，因为这些函数将游戏置于一个闭环中，直到输入被解析。
  • 调试覆盖图(The Debug Overlay)，一个游戏内的叠加视图，您可以在其中查看FPS、日志和音频窗口，以及创建您自己的自定义调试视图。

注意 尽管它们可能在某些平台上工作，但除了调试之外，不能用于任何其他用途。

注意您可以在调试器部分找到有关如何调试游戏的详细信息。

调试函数(基本)

以下函数提供了调试游戏的基本功能：

  • debug_mode 调试_模式
  • debug_event 调试_事件
  • debug_get_callstack 调试_获取_调用堆栈
  • exception_unhandled_handler 异常处理程序
  • get_integer 获取_整数
  • get_string 获取_字符串
  • show_error 显示_错误
  • show_message 显示_信息 (提示框)
  • show_question 显示_问题 (提示框)
  • show_debug_message 显示_调试_信息 (提示代码编辑器sdm快速输入)
  • show_debug_message_ext
  • show_debug_overlay 显示_调试_覆盖图
  • code_is_compiled 代码_已_编译
  • fps 帧数 (实际跑出来的帧数)
  • fps_real 实际帧数 (依据性能计算出可达到的帧数)

调试覆盖图(The Debug Overlay)

GameMaker还具有更高级的调试功能，可从调试覆盖图访问。

调试宏

GameMaker 还有几个内置宏，您可以使用它们来更轻松地调试游戏：

  • _GMLINE_ - 这是脚本编辑器中显示的行号(您可以在代码编辑器首选项中启用这些行号)。
  • _GMFILE_ - 这是使用此宏的当前代码文件的名称。这将看起来像"gml_GlobalScript_Script1" 如 Script Asset, "gml_Object_Object1_Create_0" 如 Object Events 和"gml_Room_Room1_Create" 如 Room_Creation_Code.
  • _GMFUNCTION_ - 这是使用此宏的最内层函数的名称。如果在任何函数定义之外使用，则它与 _GMFILE_ 具有相同的值，即直接在对象事件或脚本中。

如果出现上述屏幕截图，调试输出将如下所示：

gml_GlobalScript_Macros
3
show_macros
gml_GlobalScript_Macros
10
gml_GlobalScript_Macros

错误

除了这些专业函数和宏外，为了帮助您创建游戏，GameMaker 还提供了一个很长的错误消息列表，可以在编写游戏代码、编译游戏或运行游戏本身时显示。这些消息指向有问题的特定错误，并可用于查明特定对象或脚本函数中的问题，这对调试游戏的过程有很大帮助。但是，错误消息的确切含义可能并不总是很明显，因此以下部分列出了所有可能的消息，并解释了它们的含义，以使您的工作更轻松：

  • 编译器错误
  • Runner错误
