iOS 游戏选项

本节概述了可用于控制如何编译您的 iOS 游戏项目的不同选项。不同的章节是:

 

常规

常规部分的第一部分介绍了游戏的生成设置。首先，设置游戏可以运行的方向，以获得最佳用户体验。现有的选择包括:

  • 竖向
  • 竖向翻转
  • 横向
  • 横向翻转

然后您可以转到设备支持部分，该部分有下列选项可用于设置目标设备的类型:

  • iPhone or iPod Touch : iPhone 6及以上或 iPod Touch 设备
  • IPad: iPad 设备
  • Both: iPhone/iPod 和 iPad

要做的第二件事是填写有关游戏的产品信息 ，包括其显示名称、其Bundle ID 和版本信息。注意，要使游戏的最终版本正确工作，包 ID 必须采用正确的反向 URL 格式，例如: com.[公司名].[游戏名]。这里还可以提供您希望用于为 GameMaker 为您创建的最终应用程序签名的团队标识符。在此处设置它将重写在 iOS 首选项中提供的团队 ID。

接下来，可以选中或取消选中延迟主指示器输出的选项。选中此选项后，主栏总是显示为低 alpha 值，第一次滑动会唤醒主栏，第二次滑动会带您回家。如果选项未选中，那么游戏中将不显示条形图，但是第一次边缘滑动将带您回到主页。默认为取消勾选。

最后，您可以选择生成 Podfile 和 Podfile.lock 。如果使用扩展编辑器将 CocoaPods 添加到 iOS 扩展 ，则需要这些文件。文件包含特定的宏所需的Gamemaker设置任何列出的CocoaPods时，构建您的游戏。单击这些按钮将创建带有预生成宏的文件，如果您希望手动编辑这些文件，还可以打开文件资源管理器中的文件位置。还有一个按钮可以带你到下面的指南:

  • iOS and tvOS: Using Cocoa Pods

 

图形

这里您可以使用下列选项更改与游戏显示方式相关的下列详细信息:

  • 像素间插值颜色：启用线性插值，基本上是“平滑”像素。对于清晰的像素图形，它应该处于禁用状态，但如果您有漂亮的Alpha混合和平滑的边缘图形，则最好保持启用状态。默认情况下，此选项处于禁用状态。
  • 半个 iPad 1的纹理: 与最新的设备相比，iPad 1的纹理内存要少得多，所以如果你想要针对这个设备或者获得最大的兼容性，你可以使用这个选项将纹理的大小减少一半。
  • 缩放 : 在这里你可以选择缩放画布来保持设备屏幕的高宽比——在边缘添加“填充”来适应屏幕——或者你可以选择拉伸画布来适应设备屏幕。

最后，可以选择设置纹理页面的大小。默认（最兼容）大小为2048 x 2048，但您可以从256 x 256和8192 x 8192之间的任意位置进行选择。还有一个标记为预览的按钮，它将生成此平台的纹理页面，然后打开一个窗口，以便您可以看到它们的外观。如果希望了解纹理页面的结构以及防止纹理页面大于或小于所需的页面，这将非常有用。有关纹理页面的详细信息，请参阅此处。

注意：请注意，纹理页面越大，您的游戏的兼容性就越低。

 

图标

本节允许您添加各种不同的图标，需要各种 iPhone 和 iPad 设备和 iOS 应用程序商店的不同部分。这些图标应该为24位的 *.png 图像，其大小为每个图像指定。

值得注意的是， GameMaker 有一个项目图像生成器工具，可用于自动创建所有不同目标平台所需的图像。如果您使用这个工具，您应该修改所创建的图像，以确保它们是您所需要的。该工具将生成图标以及图像(见下一节)。

注意在导入较旧的项目时，可能会出现以下警告: 这是因为当前 iOS 目标版本已经更改了所需图标的格式，因为该项目以前已经加载。您将被要求重新创建所有的图标，因为它们将被替换为通用的 GameMaker 图标(您可以使用上面提到的项目图像生成器工具来快速完成)。

 

图像

本节允许您添加单独的图形作为启动屏幕(图片将在您的应用程序加载时简要显示)的每个设备方向，以及边框颜色 ，将用作背景启动屏幕，而不是设置为填充全屏幕。这些映像必须作为单个24位 * .png 图像，没有透明度并且具有适当的大小。请注意，这被认为是一个启动画面，苹果公司有一些相当严格的指导方针，这应该如何设置和使用。有关更多信息，请参见人机界面指南上的开发人员页面。

您还可以通过以下选项设置启动屏幕的显示方式:

  • 保持长宽比 : 这将保持图像与设备屏幕的长宽比，在需要的地方边缘添加“填充”。
  • 裁剪至适合 : 这将使图像与设备屏幕保持1:1，并根据需要裁剪(或填充)边缘。
  • 拉伸 : 这将拉伸图像以填充设备屏幕(并可能使显示的图像失真)。
