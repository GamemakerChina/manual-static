show_debug_overlay

此函数可用于在测试游戏时打开和关闭标准调试覆盖，默认情况下处于禁用状态。调试覆盖在实际游戏窗口本身中显示图形CPU/GPU使用栏，以及当前实际FPS值、纹理交换数量和顶点批次数量(请注意，纹理交换和顶点批次永远不会为零，并且通常会显示2或3的值，因为即使有一个空房间，没有对象GameMaker仍然必须绘制和批处理内容)。

这个条被分成几个部分，每个部分是1/60秒。如上图所示，条形由各种颜色组成：

  • 绿色-输入/输出处理(即：键盘，鼠标，游戏手柄，网络等..)
  • 红色-步骤事件的更新速度
  • 黄色-绘制事件所需的时间
  • 橙色-调试更新时间，通常只有在使用调试模块时才可见
  • 白色-GPU剩余时间，即在下一帧开始之前等待GPU完成该帧渲染所花费的时间
  • 青色-文本渲染时间
  • 灰色-每个绘制步骤清除屏幕所需的时间
  • 蓝色-运行垃圾回收器所需的时间
  • 暗红色-GPU刷新，即GPU从内存中清除图像所需的时间

使用此函数，无论是否处于调试模式，您都可以添加覆盖，通过这种方式，您可以看到游戏运行的效率，并获得有关如何使用可用资源的视觉提示，而无需同时运行调试器。

注意：此函数不适用于HTML5目标，因为它的工作方式与所有其他目标不同。

 

语法:

show_debug_overlay(enable);

参数类型 描述
enableBoolean 打开(true)或关闭(false)调试覆盖。

 

返回:

N/A

 

示例:

if global.debug
{
    show_debug_overlay(true);
}
else
{
    show_debug_overlay(false);
}

上面的代码将根据全局变量的值打开或关闭调试。
