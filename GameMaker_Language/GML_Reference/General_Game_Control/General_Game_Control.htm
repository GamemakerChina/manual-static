通用游戏控制

这些函数和变量都与游戏特定函数有关，例如加载/保存，重新启动等。

以下全局范围内置变量对于调试您的游戏很有用-例如，在出现问题的情况下，它们可以用作日志文件的输出-以及显示当前构建ID等...对用户:

  • game_id
  • game_save_id
  • game_display_name
  • game_project_name

 

以下函数用于一般的游戏控制:

  • game_end
  • game_restart
  • game_load
  • game_load_buffer
  • game_save
  • game_save_buffer
  • game_get_speed
  • game_set_speed

 

为了帮助您创建和维护游戏的高分，GameMaker创建了一个包含10个位置的全局高分数组，您可以访问、添加和更改该数组，以创建您自己的自定义高分列表，而不会有太多麻烦。此函数是游戏的本地函数，适用于所有平台，因此您可以使用这些函数以及文件处理函数轻松存储和维护基本的高分表。

存在以下处理内部高分列表的函数。

  • highscore_add
  • highscore_name
  • highscore_value
  • highscore_clear

 

GameMaker还包括一个函数，用于将屏幕上的高分作为列表绘制出来，然而，这是非常基本的，建议您使用它进行调试，而不是其他任何事情，因为制作自己的高分表很容易，并且可以让您更好地控制它的显示方式:

  • draw_highscore

 

最后，我们有一个函数，允许您为鼠标光标分配一个精灵:

  • cursor_sprite
