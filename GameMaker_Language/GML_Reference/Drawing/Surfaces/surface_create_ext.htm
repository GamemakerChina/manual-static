surface_create_ext

此函数允许您将表面附加到网页中已存在的画布元素，这意味着您可以有效地拆分游戏的各个部分，以便在页面中的各个不同位置绘制。为此，必须在游戏的*. html页面中正确定义画布元素，并使用与要创建的表面相对应的正确大小和名称。因此，您将拥有"main"画布，然后是次表面画布元素，这些元素将使用此函数指定给正确的表面。下图是如何设置具有三个画布元素的页面的示例：

页面布局可能很复杂，尤其是如果您希望所有元素都正确排列，但创建布局的硬任务完成后，您可以使用游戏选项的HTML5选项卡将此html文件添加为游戏的默认页面文件。接下来您应该做的是设置房间和视图，因为每个表面都需要与特定视图关联，以便"捕捉"游戏图像(请参阅视图变量view_surface_id[ 0...7 ])。下面的图像显示了如何设置上面画布示例的游戏房间：

最后，您将使用此函数创建表面，其名称与画布元素所使用的名称相同，大小与同一画布的大小相对应。该函数将返回表面的索引，该索引应存储在一个变量中，以便将来调用函数。首次创建表面时，它可能包含"噪波"，因为基本上它只是为此目的而放在一边的内存区域(该内存可能仍包含信息)，因此您可能希望在使用诸如draw_clear_alpha()之类的函数之前清除表面。

此函数将始终使用 surface_rgba8unorm 格式创建表面 。

注意 此函数仅适用于HTML5模块。

 

语法:

surface_create_ext(name, w, h);

参数 类型 描述
name String 要将表面链接到的画布元素的名称。
w Real 要创建的表面的宽度。
h Real 要创建的表面的高度。

 

返回:

Surface ID

 

例子:

s1 = surface_create_ext("surface1", 192, 550);
s2 = surface_create_ext("surface2", 608, 186);
view_surface_id[1] = s1;
view_surface_id[2] = s2;

上述代码创建两个不同大小的表面，将每个表面指定给不同的画布元素，然后将这些表面指定给两个视图，以便捕捉房间的正确部分。
