The Track Panel

What your sequences do is governed by the assets and parameters that have been added to the Track Panel. You can add a track to the editor by either:

  • 将资源拖到序列画布
  • 将资源拖动到轨道面板本身
  • 通过单击轨道面板底部的"添加资源" 按钮。

最后一种方法将打开资源浏览器，您可以在其中选择要添加的资源。

无论您选择哪种方法，都将为资源创建新的资源轨道。此轨道是序列中资源的主轨道，并将在关键帧表中的播放头位置为资源创建相应的资源关键帧轨道：

请注意，可以从轨道面板中通过单击末尾的彩色条来设置关键帧表中资源关键帧的颜色。这将打开拾色器，您可以在其中选择使用哪种颜色作为关键帧表中的关键轨道。

轨道编辑器中的每个资源还可以添加一组参数轨道。这些轨道控制可以随时间为轨道修改的各种不同属性(参数)。在本页下面的“添加参数轨道”一节中，详细介绍了可用的不同参数，但为了让您对此处提供的参数有一个简单的了解，下面是一个简短的列表：

  • 所有资源：位置和旋转。
  • 声音：音高和音量。
  • 序列：原点和标度。
  • Particle Systems: Colour Multiply, Origin and Scale
  • 精灵和对象：原点、比例、图像索引、图像速度和色彩倍增。
  • 文本：原点、比例、颜色倍增、字符间距、边框大小、行距和段落间距。

或者，您也可以单击“添加参数轨道” ；按钮，该按钮将打开一个菜单，显示可用不同选项：

从此菜单中选择任何选项，或直接从 IDE 录制，将在主资源轨道下创建一个或多个参数轨道，还会向关键帧表添加一个 参数键 ，颜色与主资源轨道颜色相同：

将参数轨道添加到主资源轨道后，关键帧表将有一个彩色条来显示资源轨道的长度 (以帧为单位)，并且将在希望发生参数更改的点处添加 参数关键帧 ( 关键帧)。在上图中，这些关键帧可以看作是帧上的简单"点"。当轨道具有这样的参数关键帧时，默认情况下，它们引用的参数将在显示的每个点之间进行内插 (使用  线性插值 )。例如，如果您有一个旋转参数，并且在第 0 帧将其设置为 90°，则在第 30 帧将其设置为 180°，然后在第 15 帧将其设置为 135°。通过选择参数轨道，然后在关键帧表中拖动播放头，可以实际查看值随时间的变化：

请注意，对于所有参数轨道，您可以从参数轨道上的鼠标右键 ；菜单禁用不同关键帧之间的默认线性插补，这意味着参数值之间的过渡将在给定帧上立即发生，而不是随着时间的推移逐渐发生：

除了可以选择参数轨道的过渡类型外，右键菜单还提供以下选项：

  • 剪切、复制和粘贴：这些选项允许您剪切/复制/粘贴资源或参数轨道。
  • Duplicate Track(s): This will create a duplicate of the currently selected track(s). If this is an asset track it will also duplicate the parameter tracks associated with it.
  • 删除轨道：这将删除选定的轨道。如果删除资源轨道，则与其关联的参数轨道也将被移除，但是您可以单独删除参数轨道。
  • 组 : 在轨道面板中的空白区域上使用此选项将创建一个组文件夹，然后您可以使用该文件夹来帮助将资源保持在更有序的格式中。只需单击一个资源并将其拖放到文件夹中，就可以向组中添加一个资源。如果您已经选择了一个或多个轨道，那么使用此选项不仅会创建一个组文件夹，而且还会将轨道添加到其中。
  • 重命名 : 可以使用此选项重命名任何资源轨道或组文件夹。

资源和参数轨道的一个重要特性是父轨道和组的能力。如果您将一个资源轨道拖动到轨道面板中的另一个资源轨道上，第一个资源将成为它被拖放到的资源的子资源，而这个子资源又将成为父资源。分配给父资源轨道的所有参数轨道将自动应用于子资源轨道，因此您可以使用单个位置资源轨道来移动多个精灵(例如) : 使用组也可以完成同样的操作。您可以通过单击轨道面板底部的按钮创建一个组文件夹，然后 GML 可视化您希望在组文件夹上组合在一起的资源轨道。这些资源仍然保留它们自己的参数轨道，但是您可以为组提供额外的参数轨道，组中的所有资源都将受到这些参数轨道以及它们自己的参数轨道的影响。

当使用轨道面板时，也可以使用通常的键盘快捷键，例如 :/ 和鼠标左键 以任意顺序选择一个或多个轨道， 和鼠标左键 来选择连续的轨道，还可以使用 /"X"，/ 来剪切、复制和粘贴它们 "C" 和 /"V" 组合键。

这是使用轨道编辑器的基本知识，但是下面我们将更详细地介绍每个不同部分的用途以及如何使用它们:

 

更改方向更改方向

更改方向按钮可以用于更改序列编辑器显示不同部分的方式。默认情况下，您将在顶部显示"画布视图"，在底部显示"轨道编辑器"和"关键帧表"，但这可以通过单击按钮在不同的布局中循环或单击"选项"按钮进行更改打开菜单并选择所需的布局。以下布局可用：

Split Window

使用此选项，序列编辑器将被拆分为两个单独的窗口，其中主 IDE 窗口将在整个桌面中显示画布视图，新窗口将显示关键帧表和轨道编辑器。再次单击该按钮将再次更改布局并删除额外的窗口。
Snap Top

该选项将"轨道编辑器"和"关键帧表"置于 IDE 的顶部。
Snap Right

该选项将"轨道编辑器"和"关键帧表"置于 IDE 的右侧。
Snap Bottom

该选项将"轨道编辑器"和"关键帧表"置于 IDE 的底部（这是默认视图）。
Snap Left

该选项将"轨道编辑器"和"关键帧表"置于 IDE 的左侧。

您还可以随时通过单击重置选项重置序列编辑器布局. 这将使编辑器恢复到默认状态，并重新打开所有已关闭的部分。

 

轨道搜索轨道搜索

此搜索框允许您快速搜索序列的各个轨道，以找到要编辑的轨道。在此处键入任何文本都会告诉您找到了多少带有该字符串的资源轨道，并突出显示搜索框一侧的箭头。单击这些箭头将跳到满足搜索条件的第一个和连续资源。

 

资源追踪资源追踪

添加到序列中的每个资源都将被赋予其自己的轨道，以下列出了这些轨道。如果资源轨道设置了任何参数，则这些参数也将显示在下面的参数轨道中。您可以通过单击来扩展或收缩参数轨道在轨道名称左侧的箭头图标上，您可以通过缓慢双击编辑轨道名称在轨道上，然后给它你选择的名字。请注意，您的名字没有任何限制，如果您愿意，可以使用空格、非字母数字字符并以数字开头。

在本页后面的"添加参数轨道"一节中更详细地解释了可用的不同类型的参数轨道。

 

关键点控制关键点控制

添加参数轨道后，可以向特定帧添加和删除参数关键帧，以设置该参数的值。这可以使用"记录关键点"或"删除关键点"按钮（有关详细信息，请参阅"轨道关键帧控件"一节），也可以使用参数本身上的关键点控件来执行此操作。要使用这些控件，需要将"关键帧表"中的播放头定位到要编辑的帧，然后可以单击参数轨道左侧的圆形"关键帧"图标。如果框架没有指定关键帧，则圆将为"空"，单击它将添加一个新的关键帧，并允许您编辑参数值。如果框架已经有一个关键帧，那么圆圈将被"填充"，单击它将从框架中移除关键帧：

您也会注意到，在上面的图像中，您可以使用参数关键帧标记两侧的箭头按钮快速跳过关键帧表中的关键帧，而无需手动定位播放头。

 

参数值参数值

添加参数轨道后，只需使用要更改的关键帧将"关键帧表"中的播放头设置为帧，然后使用不同的画布小部件编辑资源的位置、旋转、缩放或原点，即可编辑其中一些值。这些更改将反映在该帧的关键值中。但是，并非所有参数都可以这样编辑，或者可能不够精确，无法满足您的需要，在这些情况下，您可以通过单击轨道中的参数值并将其设置为参数类型所需的值来直接编辑参数值。

请注意，某些参数轨道可能具有"链接"图标这意味着它们可以链接在一起。这是通过单击图标启用/禁用链接来完成的，启用后，编辑一个值也会设置另一个值。

 

事件事件

以与常规对象类似的方式，序列可以具有可以运行分配给它们的某些代码的事件。代码是以脚本函数的形式分配的，该函数可以不带参数，并且在事件触发时将被调用。

通过单击添加事件按钮添加活动这将打开以下窗口：

此处列出的事件如下：

  • 创建 - 此事件发生在序列的实例首次创建时，并且是在输入房间时通过房间编辑器放置在房间中的序列中发生的第一件事。
  • 销毁 - 此事件在销毁序列的实例时执行，并将在清理事件之前运行（请参见下文）。
  • 清理 - 将在从房间中删除序列实例的任何事件之后调用此事件。因此，如果序列被破坏，如果房间结束，或者游戏结束，它将被触发，并且被设计为在序列实例出于任何原因从游戏中移除时执行任何需要执行的任务。如果您使用例如 layer_sequence_destroy() 销毁了序列实例，那么将在销毁事件之后调用此事件（请参见上文）。
  • 步开始 / 步 /} 步结束 - 步骤事件是在序列实例存在时对游戏的每一步（帧）进行检查的事件，分为三部分：开始、步骤和结束。对于大多数情况，标准步骤事件可以使用，但有时您需要对代码的运行和时间有更多的控制，因此您将获得开始和结束步骤事件，并且这些事件将始终以相同的顺序在游戏的每个步骤（帧）触发。 重要！ 如果序列被暂停，那么这些事件将 不会 被触发，当播放恢复时，它们将在序列 再次开始播放后的下一帧中 被触发。还请注意，事件的顺序不受播放头方向的影响，即使顺序向后播放，事件仍将作为 " 开始"、"步 " 和 " 结束 " 运行。
  • 异步事件 ：这相当于对象 异步系统事件 。 重要！ 如果序列暂停，则 不会 触发此事件。

可以为每个事件分配一个函数，该函数将在事件触发时调用。您可以使用通风口的输入框并单击箭头按钮来分配功能将打开要编辑的函数的脚本编辑器。您还可以单击底部的"新建"按钮，创建一个新的脚本资源，其中包含已定义并准备好填充的"样板"函数。请注意，您可以将函数名称更改为所需的任何名称，而不必使用预定义的名称，也可以删除任何不需要的函数定义。重要的是，用于序列事件的函数不能接受任何参数。

也可以使用代码添加和编辑序列事件。有关详细信息，请参阅此处。

可见/锁定开关可见/锁定开关

资源轨道和参数轨道都有一个按钮来切换轨道可见性 并锁定轨道 。如果禁用轨道的可见性，则在房间中运行序列时不会显示该轨道，测试时也不会显示在序列画布中，切换资源轨道的可见性也会切换其参数轨道的可见性。锁定轨道只意味着该轨道将无法再以任何方式编辑。请注意，不可见的轨道将在涂料表上淡入淡出，而锁定的轨道将使用覆盖绘制，因此您可以立即看到每个轨道的状态：

 

轨道颜色轨道颜色

IDE 将为您创建的每个资源轨道指定一种随机颜色，以便于在 " 关键帧表 " 上进行识别。只需单击轨道的颜色栏，然后从打开的颜色选择器窗口中选择要使用的颜色，即可随时更改此颜色。为资源轨道选择的任何颜色都将自动应用于所有关联的参数轨道，但是您可以单独编辑每个参数轨道的颜色，以便根据需要进行不同。请注意，如果随后再次更改资源轨道颜色，这将覆盖对参数轨道所做的任何更改。

 

添加参数轨道添加参数轨道

添加参数 按钮可用于将单个参数轨道添加到所选资源轨道。如本页前面所述，参数轨道可以是各种不同的类型，具体取决于与之相关的资源。下面列出了所有可用的不同参数轨道以及它们所指的内容，以及它们可以应用的资源：

  • Position: This parameter track can be added to any of the four sequence asset types (sprites, objects, sounds, sequences or particle systems) and is used to set the spacial position within the sequence for the asset. It takes an X and a Y value.
  • 旋转：与位置参数一样，旋转参数可用于 所有 序列资源，并可用于设置资源的旋转度。请记住，GameMaker 计算 逆时针旋转 ，从右侧 0°开始，因此 90°向上，180°向左，270°向下。
  • Origin: The origin parameter is available for sprite, object, sequence and particle system assets and can be used to set the X and Y position of the origin of the asset. Changing the origin will affect how the asset is positioned, rotated and scaled.
  • Scale: The scale parameter is available for sprite, object, sequence and particle system assets and is used to set the horizontal and vertical scale of the asset. This will affect how the asset is rendered in the IDE and at runtime, and is calculated as a scalar value, where 100% is the default asset size, 50% would be half size, and 200% would be double the size. Note that you have the possibility of linking the horizontal and vertical scale values by clicking the "link"  button. This means that changing one value will automatically change the other too.
  • 图像索引 和 图像速度：这些参数仅适用于 精灵 和对象 资源，可用于更改当前动画帧（有关详细信息，请参阅 image_index 上的页面）以及当前动画速度（使用 fps 设置），其工作方式与您为精灵编辑器中使用的精灵设置的值相同。
  • Colour Multiply: This parameter is only available for sprite, object and particle system assets and can be used to change the current blend colour for the asset being drawn. This is the equivalent of setting the image_blend value and any colours set here will be "multiplied" (i.e., blended) with the colour of the sprite, and a value of white (#ffffffff) indicates that no blending should take place. Note that when working with this type of track, you can set the colour by manually typing in the hex value, or you can click the colour swatch to open the GameMaker Colour Picker. Note that colours are given in ARGB format, so you can set a value for alpha using this kind of parameter track (the Colour Picker also permits you to set the alpha value). This would be the equivalent of setting the image_alpha for the sprite.
  • 音量：音量参数仅适用于 声音 资源音轨，可用于设置声音的增益（实际的声音资源增益不会改变，因为它本质上是一个发射器，用于资源音轨的序列）。体积值以百分比计算，其中 0% 为无音量，100% 为满音量。
  • 音高：音高参数仅适用于 声音 资源音轨，可用于设置声音的音高（与音量一样，实际的声音资源增益不会改变，因为它本质上是用于资源音轨的发射器）。音高值以百分比计算，100% 是声音的"自然"音高，小于 100% 将降低音高，大于 100% 将提高音高。
  • 衰减：衰减参数仅适用于声音资源轨道，目前尚未实现。
  • 有些附加参数仅适用于文本轨道。有关它们的信息请阅读序列文本。

 

轨道操作轨道操作

轨道编辑器底部的按钮是不同的轨道操作，可用于以下目的：

  • - 此按钮将打开资源浏览器，以便您选择要用作新轨道的资源。选择后，将创建资源轨道，并将帧添加到当前播放头位置的 " 关键帧表 " 中。
  • - 在轨道编辑器中选择一个或多个资源轨道后，单击此按钮将复制它们（以及它们可能包含的任何参数轨道）。
  • - 单击此按钮将创建一个特殊的 剪切蒙版 ，您可以在其中指定用作遮罩的精灵和要遮罩的精灵。有关详细信息，请参阅此处的 。
  •  - Clicking this will add a new group folder to the track editor, which can then be used to better organise tracks. You can add a track to a group by dragging the track onto the group folder and you can remove it again by dragging it out of the folder. Note that any asset that is added to a group folder will automatically be assigned the folder colour, although you can then edit the track colour for individual tracks so that it is different if required.
  • - 如您所料，单击此按钮将删除所选轨道。如果这是一个资源轨道，那么它包含的所有参数轨道也将被删除，尽管您可以在选定的参数轨道上使用它来单独删除它们，而不删除与它们关联的资源轨道。如果在空的组文件夹中使用此选项，则该文件夹将被删除，但如果文件夹中有任何轨道，则会显示警告，因为删除文件夹也会删除其中的所有轨道。
