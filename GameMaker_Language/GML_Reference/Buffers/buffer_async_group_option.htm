buffer_async_group_option

此函数为正在保存 / 加载的缓冲区组设置一些特定于平台的选项。可用的选项如下：

  • "subtitle" 或 "slottitle" - 此选项的值是一个字符串，当用户在操作系统中管理保存数据时会向用户显示该字符串。这仅在保存数据时才重要，而在再次加载时则不重要。
  • "showdialog" - 此选项的值为布尔值 true 或 false。如果设置为 true，当您保存或 加载时，GameMaker 将显示系统 UI，否则它将执行快速保存 / 快速加载，但不显示 UI。通常，仅当您支持多个保存槽以允许用户选择一个槽时，您才需要此选项，但如果您只支持每个用户一个槽，请将其设置为 false。
  • "savepadindex"-对于此选项，您需要指定正在保存或加载的玩家的pad 索引，系统将向该玩家的保存文件夹写入数据并从中读取数据。
  • "saveslotsize" - 此选项要求您指定要保存的实际大小 (以字节为单位 )( 因此您可以执行 buffer_seek 和 buffer_tell 来获取例如)。请注意，并非必须提供此值，因为所有保存都预先分配了最小空间，该空间通常随目标平台而变化。
  • 以下是一些仅适用于 PlayStation 的选项。有关信息(您需要使用您的开发人员访问账户登录)，请参阅本帮助台文章。

重要  此函数仅对 PlayStation、Xbox 和 Nintendo Switch 目标有效。在所有其他目标上，它不会采取任何行动。

 

语法:

buffer_async_group_option(option, value);

参数类型描述
optionString设置选项。
valueReal or String要设置的值(可以是实数或字符串，取决于选项)。

 

返回:

N/A

 

例子:

buffer_async_group_begin("save_folder_name");
buffer_async_group_option("showdialog", false);
buffer_async_group_option("slottitle", "Catch The Haggis Save");
buffer_async_group_option("subtitle", "All your haggis are saved here!");
save = buffer_save_async(buff, "Player_Save.sav", 0, 16384);
buffer_async_group_end();

上面的代码启动一个缓冲区组，然后设置组选项，然后设置4个文件异步保存。然后结束组定义(此时将开始保存)。
