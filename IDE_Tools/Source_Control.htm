Source Control

源代码管理(SCM) 是通过仓库和本地源处理项目的 子版本 或备份的方法的名称。基本上，SCM 解决方案是一个独立的软件包，它控制您在处理项目时维护、更改和比较项目版本的所有方面。这对那些在团队中工作并且需要能够控制谁做什么并且不担心丢失数据或进行可能需要在以后撤消的更改的人特别有用，但个人也可以从这个强大而灵活的系统中受益。

GameMaker makes use of Git for its SCM, which must be installed and set up separately before SCM can be used within GameMaker. See the Git Documentation for more specific info regarding Git.

GameMaker 将在源代码管理的菜单栏中显示一个下拉菜单，一旦为您的项目激活了源代码管理，该下拉菜单将填充以下选项 (有关详细信息，请参阅上面的设置 Git 插件一节)：

我们在这里概述了每个可用选项(上面的教程部分对其中大多数进行了更深入的说明)：

  • 创建项目仓库：此选项允许您在与项目文件相同的目录中创建本地仓库。
  • 克隆仓库：此选项允许您将仓库从源克隆到新目标。
  • 提交更改：使用此选项，您可以在项目中存放已更改的文件，然后将它们提交到仓库。
  • 推送更改：在执行提交或合并后，此选项用于将更改推送到主仓库。
  • 拉取更改：使用此选项，您可以通过从主仓库中拉取已更改的文件来更新本地仓库。
  • 查看历史记录：打开历史记录窗口，可以查看项目的所有版本历史记录，并选择回滚特定文件路径或整个修订版本。
  • 显示冲突：这将打开冲突窗口并列出项目中可能存在的任何冲突文件，允许您通过 GameMaker IDE 或使用特定的合并工具来处理它们。
