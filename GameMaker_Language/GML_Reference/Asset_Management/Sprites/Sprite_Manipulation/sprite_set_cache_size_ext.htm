sprite_set_cache_size_ext

当执行图像混合时，HTML5不能以与执行可执行文件相同的方式动态执行。因此，GameMaker必须临时保存图像的混合副本并将其加载。此函数设置在覆盖旧的精灵之前可以缓存多少个给定精灵的混合副本。默认值为4。这被应用于精灵的一个单个给定的子图像。

 

语法:

sprite_set_cache_size_ext(ind, index, max);

参数 类型 描述
ind Sprite Asset 要更改其缓存大小的精灵的索引。
index Real 要更改缓存大小的IND的子映像。
max Real 可以存储的精灵缓存副本的最大数量。

 

返回

N/A

 

例子:

sprite_set_cache_size_ext(sprite0, 0, 2);

这将把sprite0的第一个子图像的精灵缓存设置为2个副本。
