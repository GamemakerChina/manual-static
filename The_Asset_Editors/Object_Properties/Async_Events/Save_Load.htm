保存/加载

此事件将由与加载相关的某些函数的 回调 触发并将缓冲区保存到文件，以及从内存中加载或卸载音频时。事件本身将包含内置的 async_loadDS 映射，该映射将由特定功能所需的键填充。以下各节列出了这些问题。

zip_unzip_zip 和 zip_save 函数也会触发此事件。

缓冲区

当使用函数 buffer_save_async() 或 buffer_load_async() 当数据搬迁完成时，将触发异步事件。此事件将使用以下键 / 值对填充 async_load 映射

  • "id"：使用的函数返回的异步函数的ID。
  • "status"：如果数据保存/加载正确，则返回true，否则返回false。

这允许您轮询保存/加载进度并在过程完成时显示消息或更改房间等。

音频组

使用 音频组 时，您可以使用函数 从内存中加载和卸载它们 audio_group_load()audio_group_unload(). 使用加载功能时，全套音频时会触发此事件为组设置的文件已加载到内存中，并将使用以下键 / 值对填充映射：

  • "type"：这告诉我们正在调用的事件类型，并且将被"audiogroup_load"以加载音频。
  • "group_id"：将返回已加载的音频组的 ID(如 音频组编辑器 中定义的)。

当为一组加载所有音频时，将触发此事件，然后可以使用它来更改房间，或播放音乐曲目等…
