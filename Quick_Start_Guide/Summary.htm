总结

我们已经读完了这篇快速入门指南，我们希望你已经获得了足够的知识和信心来开始使用 GameMaker 来制作你自己的项目！如果您已经阅读了本指南的所有页面，那么您现在应该知道以下内容：

  • 如何创建精灵和图块集并在图像编辑器中编辑它们
  • 如何创建对象和房间
  • 对象和实例之间的区别
  • GameMaker 如何绘制东西以及如何使用绘制事件
  • 绘制事件和绘制 GUI 事件之间的区别
  • 如何在房间中移动实例
  • 基本编程概念，如变量、函数和表达式

你在很短的时间内走了很长的路，但这并不是旅程的终点。本手册有两个广泛的参考章节，涵盖了 IDE 的各个方面，以及如何使用 GML 可视化或 GML 编程，我们强烈建议您在不确定某事或想尝试新事物时回到这里：

  • IDE 参考
  • GML 可视化参考
  • GML 代码参考
  • 额外的信息

您还可以从开始页获取许多 GameMaker 教程，这些教程使用您在此处学到的核心概念来制作小而有趣的游戏，如 Space Rocks。

有关设置不同目标平台等方面的更多技术帮助，您应该查看 GameMaker 帮助台，GameMaker 还运行一系列 技术博客 ，这些博客涵盖了使用 GameMaker 的各种不同方面，从最基本的概念到最高级的技术，因此每个人都可以找到一些东西。尤其是下面这篇文章：

  • Six Resources That Taught The Pros How to Code with GameMaker

我们还想指出，GameMaker 有自己的论坛，名为 GMC(GameMakerC 社区)，这些论坛是由社区运营的，并且充满了乐于助人的成员，他们愿意放弃一些时间来帮助解决您可能遇到的任何编程或技术问题。

最后，互联网上有大量的 GameMaker 资源，所以当你陷入困境时，一定要尝试在浏览器中搜索信息或寻找教程，还有一些活跃的社区，比如 Reddit 或 Discord，你可以在那里注册和参与。

无论您做什么，我们都希望您在使用 GameMaker 时玩得开心，如果 (当！)。如果您确实制作了一款游戏并将其发布，请不要忘记给我们写信，为 GameMaker Showcase 提供相关信息！

Happy GameMaking!
