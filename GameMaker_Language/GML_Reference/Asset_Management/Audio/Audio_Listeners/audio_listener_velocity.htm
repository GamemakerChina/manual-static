audio_listener_velocity

此函数可用于提供收听者多普勒效果，并基于从给定的相对x、y和z位置解析的向量来模拟音频运动。如果收听者本身永远不会移动，或者移动不是恒定的运动，通常不需要设置这些值，但是，例如，如果您正在制作一个滚动游戏，其中玩家有一个恒定的从下到上的运动，而敌人有一个恒定的从上到下的运动， 您将将监听器和发射器速度(对于发射器，您将使用audio_emitter_velocity())设置为适当的向量，以模拟经过播放器实例时的正确多普勒效果。

注意如果您有多个监听器，应使用函数audio_listener_set_velocity(){}。

下图显示了如何为上面给出的示例游戏进行设置：

 

语法:

audio_listener_velocity(vx, vy, vz);

参数类型描述
vxReal监听器的x速度分量(默认为0)。
vyReal监听器的y速度分量(默认为0)。
vzReal监听器的z速度分量(默认为0)。

 

返回:

N/A

 

例子:

if (speed > 0)
{
    audio_listener_velocity(abs(hspeed), abs(vspeed), 0);
}

上述代码检查播放器实例速度是否超过0，如果是，则使用适当的绝对hspeed和vspeed组件来设置收听者速度。
