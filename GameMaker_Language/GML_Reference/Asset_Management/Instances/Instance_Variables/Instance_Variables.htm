实例变量

创建新对象时，它将带有某些已用默认值初始化的变量。这些变量我们称为实例变量，因为它们将为您在游戏室中放置对象的每个实例创建，一旦创建实例，这些变量的值对于该实例是唯一的，并且只该实例。这些变量中，有些将使用很多，如x和y，而其他变量则不太需要，这取决于您想做什么，但通常它们非常有用，在可能的情况下，您应该考虑使用它们，而不是创建自己的自定义变量。

大多数实例变量都可以设置和读取，允许您通过调整某个变量的值来更改实例的属性和行为，例如，只需将visible内置变量设置为false即可防止实例绘制。 下面可以找到为游戏中所有对象的所有实例初始化的不同变量。

重要 如果实例在序列中，当序列在开始播放后更新每个步骤时，将覆盖这些变量，如 image_xscale / image_yscale / image_angle / x / y。

 

这些变量处理一般实例属性：

  • id
  • visible
  • solid
  • persistent
  • depth
  • layer
  • alarm

实例还具有一个通用变量，或者更确切地说，方法，可以定义该变量以更改此实例转换为字符串的方式。

  • toString 方法

这些变量处理实例位置和移动：

  • direction
  • friction
  • gravity
  • gravity_direction
  • hspeed
  • vspeed
  • speed
  • xstart
  • ystart
  • x
  • y
  • xprevious
  • yprevious

 

对象属性

以下变量保存创建实例的对象的唯一索引ID：

  • object_index

 

这些变量都与分配给实例的精灵相关，可用于更改所绘制的内容和方式：

  • sprite_index
  • sprite_width
  • sprite_height
  • sprite_xoffset
  • sprite_yoffset
  • image_alpha
  • image_angle
  • image_blend
  • image_index
  • image_number
  • image_speed
  • image_xscale
  • image_yscale

 

这些变量处理碰撞掩码：

  • mask_index
  • bbox_bottom
  • bbox_left
  • bbox_right
  • bbox_top

 

这些变量与路径以及实例与路径的交互方式(如果已分配)相关：

  • path_index
  • path_position
  • path_positionprevious
  • path_speed
  • path_scale
  • path_orientation
  • path_endaction

 

这些变量用于将实例设置为使用时间轴：

  • timeline_index
  • timeline_running
  • timeline_speed
  • timeline_position
  • timeline_loop

 

序列

以下变量存储有关可能在任何给定时间控制实例的序列的信息：

  • in_sequence
  • sequence_instance
