编码和哈希

在处理外部文件时，最终用户始终有可能打开并更改它们包含的信息，从而更改您的游戏。这可能会导致游戏元素损坏或在线分数出现欺诈 (例如)，因此 GameMaker 为您提供了一些基本的 编码 函数以及执行 哈希 检查的函数对字符串和文件进行检查，以确保它们在使用之前保持完整性。还提供了一些用于编码和解码 JSON 格式字符串以及创建 ZIP 文件的函数。

注意 编码不是加密！Base64 编码使文件无法用肉眼读取，并且需要用户代表进行解码，但它不能保证不被黑客攻击。建议您将这些函数与您自己的加密混合使用 (GameMaker 的加密和脚本函数在互联网上有很多种形式)。

  • json_encode
  • json_decode
  • json_stringify
  • json_parse
  • base64_encode
  • base64_decode
  • md5_string_utf8
  • md5_string_unicode
  • md5_file
  • sha1_string_utf8
  • sha1_string_unicode
  • sha1_file
  • load_csv

 

提供以下函数用于处理 ZIP 文件：

  • zip_unzip
  • zip_unzip_async
  • zip_create
  • zip_add_file
  • zip_save

 

值得注意的是，还有许多与编码和哈希 缓冲区 相关的函数，您可以在以下链接中找到它们:

  • buffer_md5
  • buffer_sha1
  • buffer_crc32
  • buffer_base64_encode
  • buffer_base64_decode
  • buffer_base64_decode_ext
