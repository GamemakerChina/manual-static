audio_start_recording

此函数将从已索引的录音器源开始录制音频。您可以使用函数audio_get_recorder_count()获得录音机源数，开始录制后，音频将存储在临时缓冲区中，并开始触发音频录制异步事件。 此事件会在记录的每个步骤触发，并将在变量async_load中创建具有以下键/值对的特殊DS map：

  • "buffer_id"可用于检索音频数据的临时缓冲区的 ID
  • "channel_index"记录通道索引(由函数返回)，此数据来自
  • "data_len"您接收到的数据长度(以字节为单位)

请注意，在触发异步事件后，临时缓冲区中的音频将被擦除，因此您应该将其数据复制到先前创建的自定义缓冲区中。

注意大多数平台支持以某种形式录制音频，但这并不意味着所有设备都会允许录制，即使平台允许，因此您应始终检查audio_get_recorder_count()函数返回大于0的值，以验证在使用其余录制函数之前录音设备是否可用。

 

语法:

audio_start_recording(recorder_index);

参数 类型 描述
recorder_index Real 要使用的记录器源的索引。

 

返回:

Buffer ID

 

例子:

audio_record = audio_start_recording(0);

上面的代码从记录器源0开始记录，将记录的通道索引存储在变量"音频_记录"中，以便在异步音频记录事件中使用。
