part_type_sprite

此函数将粒子类型设置为使用游戏资源中的自定义精灵。

注意使用part_type_shape可使用内置粒子形状之一。

注意此功能可用于更改粒子类型的现有粒子的精灵。

用法注释

  • 您可以选择精灵，然后告诉GameMaker使用精灵并设置动画。
  • 如果您选择动画，您也可以选择拉伸动画;即，以一定的间隔更改子图像，以便动画将在创建的每个粒子的生命周期的开始和结束时开始和结束(因此，如果将生命周期设置为最小15步和最大30步，则无论生命周期具有什么值，动画都将适合，对于较短的生命周期，动画运行得更快，对于较长的生命周期，动画运行得更慢)。
  • 您还可以选择随机子图像，以便如果精灵未设置动画，则会随机选择不同的图像来创建精灵，或者如果粒子已设置动画，则会在随机点开始动画。
  • 如果random为false，可以使用part_type_subimage更改子画面，以创建粒子或启动动画。

 

语法:

part_type_sprite(ind, sprite, animate, stretch, random);

参数 类型 描述
ind Particle Type ID 要更改的粒子类型的索引
sprite Sprite Asset 要设置粒子类型的精灵的索引
animate Boolean 是否遵循精灵的动画正常(true)或不(false)
stretch Boolean 是否拉伸精灵的动画以匹配粒子寿命(true)或(false)，使用part_type_life设置
random Boolean 开始时是否选择随机子图(true)或不选择(false)

 

返回:

N/A

 

例子:

global.p1 = part_type_create();
part_type_sprite(global.p1 , spr_Coins, 1, 0, 0);
part_type_size(global.p1, 1, 3, 0, 0);
part_type_scale(global.p1, 1, 1);
part_type_colour1(global.p1, c_white);
part_type_alpha2(global.p1, 1, 0);
part_type_speed(global.p1, 2, 4, 0, 0);
part_type_direction(global.p1, 0, 180, 0, 0);
part_type_gravity(global.p1, 0.20, 270);
part_type_orientation(global.p1, 0, 0, 0, 0, 1);
part_type_blend(global.p1, 1);
part_type_life(global.p1, 15, 60);

上述代码将全局变量"p1"中索引的粒子类型设置为spr_Coins中索引的精灵的形状，然后设置各种其他粒子属性。
