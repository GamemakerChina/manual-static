script_execute

此函数最初设计用于以动态方式调用脚本资源，但是引入用户定义的函数后，现在可以以两种方式使用：

  • 第一种方式是支持传统脚本，其中每个脚本都是单个函数，脚本的标题也是函数的名称。以这种方式使用，它将像以前一样工作，它将运行用户创建的脚本资源，并将给定的参数传递到其中。请注意，本例中实际调用的不是脚本，而是脚本中同名的脚本函数。
  • 第二种方式是从包含多个函数的脚本资源中调用函数。在这种情况下，您将提供函数名称及其所需的参数，而不是脚本资源名称。请注意，这只适用于脚本函数，不适用于方法，因为只有脚本函数具有分配给它们的脚本ID。 对于方法，使用method_call()。

注意 虽然此函数主要设计用于旧式支持，但它并不受欢迎，因为它在网络和远程过程调用情况下仍有其位置。还值得注意的是，该函数很慢，使用它可能会对性能产生不利影响。

 

语法:

script_execute(scr, arg0, arg1, arg2..., etc...);

参数 类型 描述
scr Script Asset or Script Function 要调用的函数/脚本
arg0, arg1, arg2..., etc... Any 要传递给函数/脚本的不同参数

 

返回:

Any (Will depend on the return value from the script/function being called)

 

例子:

script_execute(choose(move_up, move_down, move_left, move_right), irandom(5));

上面的示例代码将使用 script_execute 来选择四个用户定义函数中的一个 (在本例中与移动相关)，并将一个随机整数作为第一个参数传递给它。
