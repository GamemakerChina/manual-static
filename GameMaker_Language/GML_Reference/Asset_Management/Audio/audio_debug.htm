audio_debug

此函数可用于显示有关音频系统的调试信息，true打开它，false关闭它。以这种方式调试音频时，叠加将显示以下信息：

  • Handle - OpenAL 源句柄(每个播放的声音都独一无二)
  • Data Handle - OpenAL缓冲区句柄(为具有相同源句柄的多个声音显示相同的值)
  • Processed Buffers - 播放了多少缓冲区
  • Queued Buffers - 有多少缓冲区已排队(且仍在等待播放)
  • Gain - 音频的增益值(0- 65536)
  • Play Offset - 样本位置(当前播放位置)到当前缓冲区
  • Queued Buffers Bars - 已处理/排队缓冲区的小方框图，其中播放偏移(当前位置)绘制为蓝线

注意 对于同步的音频组，您还可以使用函数 audio_sync_group_debug。

注意 此函数在HTML5目标平台上不起作用。

 

语法:

audio_debug(enable);

参数 类型 描述
enable Boolean 启用(设置为true)或禁用(false)音频调试。

 

返回:

N/A

 

例子:

if (debug_mode)
{
    audio_debug(true);
}
else
{
    audio_debug(false);
}

根据游戏是否在调试模式下运行，上述代码将打开或关闭音频调试叠加。
