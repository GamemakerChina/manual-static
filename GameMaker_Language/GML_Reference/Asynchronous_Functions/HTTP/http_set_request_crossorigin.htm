http_set_request_crossorigin

通过该函数，您可以设置从文件加载图片时使用的跨域类型 (使用 sprite_add())，或者向服务器发送自定义的 HTTP 请求 (例如使用 http_post_string())。该函数仅适用于 HTML5 平台。

当设置为 "use-credentials" 时，您不再需要给出要加载的精灵的绝对路径，而是给出一个 相对路径 (如下例所示)。

默认情况下跨域类型设置为 "anonymous"。

 

语法:

http_set_request_crossorigin(origin_type);

参数 类型 描述
origin_type String 要使用的跨域类型(字符串)

 

返回:

N/A

 

例子:

if string_lower(http_get_request_crossorigin()) != "use-credentials"
{
    http_set_request_crossorigin("use-credentials");
}
sprite_add("sprites/player_outfit_1.png", 0, false, false, 0, 0);

上面的代码将首先检查当前设置的跨域类型，如果它不是“use-credentials(使用凭据)”，那么它被设置为“use-credentials”，然后从文件中添加一个精灵。
