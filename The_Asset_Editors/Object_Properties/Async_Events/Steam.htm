Steam

此事件由一些Steam 扩展函数的回调触发，并返回存储在变量 async_load 中的DS Map，包含不同的键/值对，具体取决于触发该事件的函数的回调。映射将始终包含键 "event_type"，您可以对其进行解析，以找到触发事件的函数类型，并更改所需的代码以适应它。

任何触发此事件的函数都会返回一个唯一的异步 ID 值，该值应该存储在一个变量中并进行检查，因为 async_load 映射将始终包含一个 "id" 键，然后您可以解析该键并确保您正在响应正确的事件。事实上，映射将始终包含以下键，而不管用于生成异步响应的 Steam 函数是什么：

  • "id"- 触发事件的函数返回的 URL ID。
  • "result"- 操作的结果 (一个真实的值)。这将是扩展常量 ugc_result_success 或其他真实的数字。您应该检查此常量以确保调用成功，如果返回了另一个值，则说明某些操作不正确。返回的其余可能值是可能的 EResult 之一由 Steamworks API 定义的值。有关完整列表，请参见 EResult。
  • "event_type"- 表示事件类型的字符串 (请参阅特定函数的文档以了解其事件类型)
