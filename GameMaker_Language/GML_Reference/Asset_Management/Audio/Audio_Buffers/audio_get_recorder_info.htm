audio_get_recorder_info

此函数将返回DS map，其中包含有关给定记录器源索引的信息。您可以使用函数audio_get_recorder_count()来查找有多少录音机源可用，返回的映射将包含以下键/值对：

  • "name"用于描述设备的名称
  • "index}"-用于记录的索引
  • "data_format"格式数据将返回(当前始终为buffer_s16，但将来可能支持其他格式)
  • "sample_rate"返回数据的采样速率(单位：Hz)(当前被箝位至16000Hz，但以后可能会改变)
  • "channels"常量audio_mono(将来可能支持立体声和3D的更多常量)

请注意，虽然该函数为您创建了一个DS map，但它不会在以后再次删除它，因此当不再需要它时，您应该自己销毁该映射，以防止任何内存泄漏。

注意大多数平台支持以某种形式录制音频，但这并不意味着所有设备都会允许录制，即使平台允许，因此您应始终检查audio_get_recorder_count()函数返回大于0的值，以验证在使用其余录制函数之前录音设备是否可用。

 

语法:

audio_get_recorder_info(recorder_index);

参数 类型 描述
recorder_index Real 要从中获取信息的记录器源的索引。

 

返回:

DS Map

 

例子:

len = async_load[? "data_len"];
audio_buff = buffer_create(len, buffer_fast, 1);
buffer_copy(async_load[? "buffer_id"], 0, len, buff, 0);
audio_queue_sound(audio_queue, audio_buff, 0, len);
audio_play_sound(audio_queue, 10, 0);

上述代码将在异步音频记录事件中调用，并将一些记录数据分配给缓冲器，然后将其添加到音频队列中。然后播放。
