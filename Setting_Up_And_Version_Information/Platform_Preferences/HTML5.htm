HTML5 首选项

HTML5 首选项有以下选项：

  • 混淆：是否应在调试模式下对 HTML5 运行器代码进行混淆处理。
  • 美化打印：游戏代码是否需要美化打印(格式化)。
  • 删除未使用的函数：是否应从最终JavaScript代码中删除未使用的函数(删除未使用的内置GameMaker函数和自定义GML函数)。启用此选项可减少运行时大小。
  • 启用前缀：此选项在缩小的代码中使用obf_作为标识符的前缀，而不是将它们重命名为短标识符，从而使调试时更容易查找内容。
  • 启用条件编译：此选项启用运行时代码的条件编译，这将删除游戏未使用的部分。它可以与删除未使用的函数一起使用，以获得最佳效果。
  • 保持JSDoc：这将在游戏代码中启用或禁用JSDoc注释。
  • 详细：在为 HTML5 生成时启用或禁用输出日志中的详细日志记录。
  • 默认打包类型：在构建最终的 HTML5 可执行包时，您可以选择打包为松散文件-这将只是在文件夹中生成所有必需的文件-或者打包为 ZIP-将创建一个压缩的 ZIP 文件，其中将包含游戏和任何其他文件。默认情况下，您可以选择在编译时选择这两个选项中的哪一个，但您可以将此首选项设置为始终使用其中之一。请注意，当此选项设置为显示对话框时，在编译时弹出的实际对话框窗口中，选择"记住打包选项"会将此首选项设置为您选择的选项。
  • 默认 Web 服务器端口：这允许您更改 HTML5 Web 服务器的默认端口。默认值为 51264。
  • 默认 Web 服务器地址：您可以在此处更改 HTML5 Web 服务器的默认 Web 服务器地址。 默认值为127.0.0.1。
  • 默认 Web 服务器允许的 IP：您可以在此处添加 Web 服务器的任何允许 IP，这些 IP 将被设置为通过目标管理器添加的所有设备的默认 IP。
  • HTML5 Runner 路径：您可以在此处更改 HTML5 Runner 的路径，以使用自定义版本的 Runner。缺省值为 ${html5_runner_path}，它指向通过 IDE 下载的 HTML5 运行器。

重要 使用特定目标设备(即浏览器)时，将使用为该特定设备设置的IP和端口值，而不使用这些首选项。检查设备管理器 以查看您的设备列表及其详细信息。
