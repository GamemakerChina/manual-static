alarm

此一维数组用于获取实例可能具有的任何报警的当前值，或者可用于设置这些报警。对象的每个实例中都内置了十二个报警，每个实例都有自己的事件，当该变量达到0时，这些事件将运行。

在每个步骤的开始、在开始步骤事件之后但在主步骤事件之前，以及在为帧呈现任何内容之前，对警报进行倒计时。这意味着，如果在BEGIN STEP事件中将警报设置为1，则其事件将在同一帧中运行(因为在BEGIN STEP完成后，警报将递减计数到0)，但如果在STEP事件中将警报设置为1，则它将在下一帧中运行。

需要注意的是，当警报达到0(尽管事件已被触发)时，警报尚未结束，作为下一步，它将下降到-1，因此，如果您出于任何原因需要停止报警，应将其数组值设置为-1而不是0。报警时间以游戏步数计算，30表示30步，60表示60步等...

注意没有动作或代码的报警将不会运行。然而，即使只有一个注释，没有代码，警报也会倒计时。

您可以直接设置报警数组(也可以直接使用该数组来检索当前报警值)，但在某些情况下这不是最合适的方法，因此您也可以使用函数alarm_set()设置报警，使用函数alarm_get()来获取报警值。

 

语法:

alarm[val];

 

返回:

Real (-1 if the alarm isn't running)

 

例子:

if canshoot
{
    if keyboard_check_pressed(vk_space)
    {
        canshoot = false;
        alarm[0] = room_speed;
        instance_create_layer(x, y, "Bullets", obj_Bullet);
    }
}

上面的代码检查一个变量，然后如果它为真，则检查按键，如果它为真，则最终将变量设置为假，设置警报(其中它将再次设置为真)并创建一个实例。
