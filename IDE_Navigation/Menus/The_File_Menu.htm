文件菜单

在文件菜单中，您可以选择更改、保存或导入项目以及设置 GameMaker IDE 首选项。可用选项包括：

  • 新建项目 - 用于在 GameMaker 中从头创建新项目。它将打开"保存"对话框，并提示您选择要将新项目保存到的位置。一旦您为项目指定了名称和保存位置，您将被带到一个新的、空白的工作区以开始处理它。项目文件以文件扩展名 *.yyp 保存。
  • 打开项目 - 这将打开文件资源管理器，以便您选择要加载的先前创建的 *.yyz 项目文件。请注意，在导入时，如果项目是使用低于 2.3.0 的版本创建的，则您将收到以下警告消息：如果您选择继续，则将打开一个文件资源管理器，并且您可以为项目指定更新的名称和位置以保存它，这样原始项目将不会被覆盖。一旦项目转换为新的 2.3.0+ 格式，它将在 IDE 中打开，您将看到一条消息，提示转换已成功。您应该继续检查转换过程生成的兼容性报告，并在尝试运行或编译项目之前查看是否有需要修复的内容。请注意，如果您尝试打开使用 2.3.0 之前的版本创建的项目时出现重复错误，则您应该联系 YoYo Games (使用帮助菜单中的"报告错误"选项)，并提供您试图转换的原始 *.yyp 文件。
  • 导入项目 - 您可以在此处导入以前导出为 *.yyz 存档的项目(请参阅下面的"导出项目")，或者-如果您有使用传统 GameMaker:Studio 1.4 制作的项目-您可以选择此选项从该产品线导入 *.gmx 或 *.gmz 文件。与打开项目一样，如果 *.yyz 是在 2.3.0 更新之前使用 GameMaker 版本创建的，那么它将需要转换，因为项目文件格式随着该更新而更改。

注意 要将以前的 GameMaker 项目导入到 GameMaker 中，它必须是使用1.4版本创建的，即使这样，它在导入时也可能无法直接工作。GameMaker 和 GameMaker:Studio 1.4 之间有很多不同之处，因此您的旧项目可能需要进行一些调整才能运行。请参阅 GameMaker 知识库中的文章将GM:S 1.4游戏移植到GameMaker，以及手册中介绍过时功能和兼容性脚本的章节。

重要! 导入包含字体资源的 *.yyz 项目可能需要您通过删除/重新添加字体或在字体编辑器中更改字体然后重新设置来重新生成字体。这是因为现有字体信息在导出时由 *.yyz 保留，并且在以后导入或在另一台计算机上导入时，项目可能尚未安装所需的字体(或可能具有相同名称的不同字体)。仅当您看到导入后字体呈现方式有问题时，才应执行此操作。

  • 最近的项目 - 您可以使用此选项查看最近使用的项目。选择它将重新打开起始页(如果已将其关闭)，您可以从该列表中选择最近使用过的项目。
  • 保存项目 - 这会将您的游戏保存到您在创建项目时定义的位置(或从中打开它的位置)。
  • 将项目另存为 - 与上述相同，只是您指定了项目的位置和新名称。
  • 导出项目 - 这允许您将项目导出为文件或模板：
    • YYZ - 这会将项目导出为单个 *.yyz 文件，并且对于向用户发送副本或保存备份非常有用。
    • 作为模板 - 这将允许您将项目导出为模板，以后可以将其用作起始页上其他项目的基础。选择此选项将显示以下提示：
      您应首先单击"编辑模板信息"以确保您的模板信息正确，然后在第二次打开此提示时选择"导出"以将其另存为模板。然后，您将在开始屏幕上的模板菜单下看到它。

      您创建的任何模板都将绑定到您登录时使用的账户。如果您稍后通过不同的账户登录，您将无法访问相同的自定义模板。
  • 新建 IDE - 选择此选项后，将打开一个新的 IDE 实例供您使用 (它将在起始页上打开)。
  • 首选项 - 选择此选项将进入首选项窗口，您可以在其中设置 GameMaker 的外观和体验。
  • 退出 - 退出 GameMaker 到桌面。
