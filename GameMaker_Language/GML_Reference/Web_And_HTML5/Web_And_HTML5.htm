Web 和 HTML5

GameMaker 提供了与互联网或服务器交互的各种函数，可以在所有可用的目标平台上使用。这些函数是:

  • url_open
  • url_open_ext
  • url_open_full
  • url_get_domain
  • http_request
  • http_get
  • http_get_file
  • http_post_string

 

除了上述列出的一般函数，HTML5目标还有一些专门为目标游戏设计的额外网页函数:

  • browser_height
  • browser_width
  • browser_input_capture
  • webgl_enabled
  • clickable_exists
  • clickable_add
  • clickable_add_ext
  • clickable_change
  • clickable_change_ext
  • clickable_set_style
  • clickable_delete
  • analytics_event
  • analytics_event_ext
  • os_browser
  • http_get_request_crossorigin
  • http_set_request_crossorigin

 

除了HTML5目标模块的这些函数，还有一个特殊的JavaScript变量，当html5game文件夹的内容存储在一个完全不同的地方(URL)时，可以使用这个变量。通常情况下，这个路径是硬编码在JavaScript文件中，与调用JavaScript的位置相关，但你可以在调用游戏脚本之前设置变量g_GameMakerHTML5Dir，这样游戏就会从新定义的位置运行。例如:

<script>var g_GameMakerHTML5Dir = "http://mygamealternatedirectory.com/";</script>

同样值得注意的是，您可以使用前缀 gmcallback_ 标记脚本函数，这意味着在为HTML5编译项目时，它们不会被混淆。这在使用上面列出的clickable_*函数时非常有用(更多信息，请参阅这里)。
