Plugin Preferences

插件首选项用于控制已添加到 GameMaker IDE中的任何额外插件。默认情况下，这里有一个专门介绍源代码管理插件的部分，用于集成 GameMaker 项目和 Git。

源代码管理 (Git)源代码管理 (Git)

本节介绍使用 Git SCM 插件与 GameMaker 配合使用所需的首选项。

  • Git 可执行文件路径：这是 GameMaker 用于所有源代码管理命令的 Git 可执行文件的位置。如上图所示，它将位于 Git 安装的 cmd 文件夹中。

在此下面，您可以使用以下设置为源代码管理设置合并工具和比较工具(可选)：

  • 合并(工具位置)：在此指定 Git 合并工具的完整文件路径。
  • 合并(工具选项)：在此字段中，您可以添加在使用合并工具时要运行的任何额外命令。
  • 比较(工具位置)：在此指定 Git 比较工具的完整文件路径。
  • 比较(工具选项)：在此字段中，您可以添加在使用比较工具时要运行的任何额外命令。

最后，还有两个选项会影响如何在新项目中设置 Git：

  • 将 .git 默认配置文件添加到新项目：启用此设置后，GameMaker 将向您创建的新项目添加 .gitignore 和 .gittributes文件。默认情况下，它处于启用状态。
  • 将 .git 默认配置文件添加到导入的项目：启用此设置后，GameMaker 会将 .gitignore 和 .gittributes文件添加到您从.yyz 文件导入的项目中。默认情况下，它处于启用状态。
