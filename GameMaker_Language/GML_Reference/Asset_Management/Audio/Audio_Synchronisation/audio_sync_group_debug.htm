audio_sync_group_debug

此函数显示有关任何给定同步组的调试信息(组索引是使用函数audio_create_sync_group创建组时返回的值)。您可以使用值 -1 来调用该函数来关闭覆盖。

调试音频同步组时，覆盖将显示：

  • 正在调试哪个同步组
  • 里面有多少个频道？
  • 同步组失败了多少次(这意味着它尝试做了太多的工作)
  • 频道列表
    • 红色表示此通道当前未执行任何工作
    • 黄色表示频道处于初始状态(尚未解码或播放)
    • 绿色表示音轨当前正在搜索/解码
    • 紫色表示通道已用完数据
  • 显示解码线程正在执行的工作的性能分析栏
    • 蓝色表示第一次读取文件
    • 红色表示线程正在读取文件，以找出从哪里读取
    • 绿色表示线程正在解码数据
    • 紫色表示线程在需要工作之前处于休眠状态

请注意，对于音频的一般调试，您还可以使用函数audio_debug。

注意此函数不适用于HTML5目标平台。

 

语法:

audio_sync_group_debug(group_index);

参数 类型 描述
group_index Audio Sync Group ID 要调试的组索引，或-1关闭调试。

 

返回:

N/A

 

例子:

if (debug_mode)
{
    audio_sync_group_debug(sg);
}
else
{
    audio_sync_group_debug(-1);
}

The above code will switch on or off the audio sync group debug overlay showing the sync group indexed in the variable sg depending on whether the game is run in debug mode or not.
