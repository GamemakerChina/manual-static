math_set_epsilon

精度用于确定两个受舍入误差影响的数字是否足够接近，从而被视为"相等"。默认精度为 0.00001。

这在处理浮点数学时很有用，因为它可以减少使某些运算返回的值看起来不正确或与我们预期相反的"舍入误差"。例如，我们可能已经向精灵的图像索引添加了一个值，并期望结果是一个整数(5)，但由于浮点数学的性质，实际的最终值最终类似于 5.000002，因此当我们进行以下检查时：

if image_index == 5
{
    //do something
}

代码的行为与预期不符，并返回 false。但是，当精度设置为 0.00001 时，image_index 值将舍入为最接近的实数，即原始值的 +/-0.000001，从而使上述比较返回 true。

根据运行游戏的平台的不同，浮点计算的行为可能会有所不同。默认的精度为跨平台舍入误差提供了安全的范围 0.00001。

精度值将用于以下所有整数运算符：

  • < : 小于
  • > : 大于
  • == : 等于
  • <= : 小于或等于
  • >= : 大于或等于
  • != : 不等于

请注意，将精度值设置为 0 将禁用所有舍入，而将值设置为 1 将产生错误。

 

语法:

math_set_epsilon(epsilon);

参数 类型 描述
epsilon Real 新的精度值(从 0 到 0.999999999)。

 

返回:

Real

 

例子:

math_set_epsilon(0.0001);

这将为所有进一步的浮点运算设置精度值。
